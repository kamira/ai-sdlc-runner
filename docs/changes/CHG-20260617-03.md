# CHG-20260617-03 — Multi-panel execution dashboard (curses)

- Project: ai-sdlc-runner
- Date: 2026-06-17
- Type: new feature
- Proposed by: Harutsuki (wants panels: execution log / verification results / status; agent behavior log with a menu toggle for per-agent tabs vs one consolidated panel)
- Implemented by: I1 (lead implementer, this session)
- Risk: low  (additive presentation layer; reads state/ACC/git and consumes orchestrator events; no governance logic, no contract surface, run behavior opt-in)
- Autonomy: (none — default halt policy applies)
- Related: CHG-20260617-01 (orchestrator/state), CHG-20260617-02 (tui); FR-10/FR-11

## Motivation
Give the runner an at-a-glance, real-time view while it drives a project, and a readable snapshot
afterward. Per the user's refinement, **branch + progress + current status are merged into one
"Status" panel**, and the reference screenshot is only a layout preview (not an HTML target) — so the
dashboard is a **terminal curses** view, consistent with the runner's stdlib-only, pure-backend stance.

## Panels
1. **狀態 / Status** — git branch + dirty flag (分支情況), stage progress bar N/4 (完成進度), current stage + per-project contract lock (目前狀態).
2. **執行日誌 / Execution log** — stage transitions and gate decisions (AUTO/HALT with reason).
3. **檢驗結果 / Verification** — acceptance reports found under `docs/acceptance/` + the latest V1 conclusion / halt.
4. **agent 行為日誌 / Agent log** — per-agent dispatch & results; **default consolidated** in one chronological panel, **menu-togglable to tabbed-per-agent**.

## Decisions & trade-offs
| Decision | Options | Why this choice |
|----------|---------|-----------------|
| Render target | curses TUI vs HTML | User: screenshot is just a preview, not HTML → terminal curses (zero-dep, matches the rest of the CLI) |
| Panel grouping | 5 separate vs merge branch/progress/status | User: merge branch+progress+status into one Status panel |
| Data sourcing | push-only vs read state/ACC/git | Live events from the orchestrator **plus** reading saved `state.json`/`.sdlc-lock.json`/ACC/git, so `runner dashboard` works post-hoc too |
| Orchestrator coupling | dashboard calls into orchestrator vs event callback | Optional `on_event` callback keeps orchestrator decoupled and backward-compatible |
| Live `run` output | always vs opt-in `--dashboard` | Opt-in so existing `run` output stays identical (backward compatibility) |
| Agent-log default | tabbed vs merged | User: default merged (consolidated); toggle to tabbed via menu / `--agent-view` |

## Modification Guide
### Goal
Add a presentation layer that renders four panels from orchestrator events + saved state/git, with an
agent-log view toggle. No new governance behavior. Map to FR-10/FR-11 (state/progress are the source).

### Impact scope
| Affected item | Type | Change | Knock-on effect |

### Modification steps
