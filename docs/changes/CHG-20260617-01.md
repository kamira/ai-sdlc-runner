## Motivation
Build the first `ai-sdlc-runner`: an external Python orchestrator that drives the ai-sdlc skill's
four-stage loop with gated halt-points. The runner references — never copies — the skill, reads all
governance logic from it, and locks the contract at major.minor per governed project. Per build-guide
§8 the first runner is built through the ai-sdlc four stages (human-in-the-loop), not by itself.

## Decisions & trade-offs
| Decision | Options | Why this choice |
|----------|---------|-----------------|
| Version detection from SKILL.md file | git tag vs file | User decision: file detection makes a missing/wrong tag a contract-version mismatch, not silent drift |
| Reference skill via submodule | submodule vs copy | One-way dependency, no drift; copying forbidden (§7) |
| Call skill scripts via subprocess | re-implement vs call | Skill is single source of truth (§1.3) |
| Lock major.minor, patch-permissive | full version vs major.minor | Patches don't force migrate; minor/major do (§5) |
| Validating migrate | force vs validate | Upgrade only if all docs re-parse (§5) |
| V1 tools exclude `Agent` | discipline vs tool-lock | Mechanical guarantee (§1.6) |
| Stdlib-only, PyYAML optional | hard dep vs tiny reader | Thin, low-dependency driver |

## Modification Guide
### Goal
Create the runner package, config, submodule scaffolding, README, and contract tests per
build-guide §2/§3, satisfying FR-1..FR-14.

### Impact scope
| Affected item | Type | Change | Knock-on effect |
|---------------|------|--------|-----------------|
| `src/ai_sdlc_runner/*` | directory/logical/design | New package (cli/contract/agents/gates/state/orchestrator) | Defines the runner's whole surface |
| `config/runner.yaml` | data | New runtime config | Limits probed at startup |

### Modification steps
1. Write `config/runner.yaml`, `pyproject.toml`, `.gitmodules` (done in Stage 0).
2. Implement `contract.py` (version read, contract_key, resolve_contract, migrate).
3. Implement `gates.py` (subprocess to halt_gate.py / cross_repo_check.py).
4. Implement `agents.py` (parse role table; spawn with role allowlist; V1 excludes Agent).
5. Implement `state.py` (checkpoint/resume).
6. Implement `orchestrator.py` (four-stage sequential loop + shallow fan-out) and `cli.py` (run/migrate/status).
7. Write `README.md` and `tests/test_contract.py`.
