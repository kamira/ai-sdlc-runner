# CHG-20260617-05 — Offline local skill store (vendored v1.0.0 + v1.1.0)

- Project: ai-sdlc-runner
- Date: 2026-06-18
- Type: refactor / new feature (skill sourcing)
- Proposed by: Harutsuki (skill must NOT be fetched online; copy v1.0.0 and v1.1.0 into a folder as the primary store)
- Implemented by: I1 (lead implementer, this session)
- Risk: medium  (changes how the skill is sourced; deliberately relaxes a build-guide guardrail — see override note)
- Autonomy: (none — default halt policy applies)
- Related: CHG-20260617-01 (submodule), CHG-20260617-04 (update detection); build-guide §1.2, §7

## ⚠ Deliberate guardrail override (user-approved, human-in-the-loop)
The build guide's §1.2 ("引用而非複製") and §7 ("❌ 把任何 skill markdown 複製進 runner") forbid copying
the skill into the runner; the intended mechanism is a git submodule pinned to a tag. **The user has
explicitly chosen to vendor the skill versions into the runner for offline use.** Per the ai-sdlc
override rule ("flag the risk, then follow the user's decision"), this CHG records the relaxation:

- Risk relaxed: single-source-of-truth via submodule → a local copy now lives in the runner repo.
- Mitigations kept: (1) the submodule config is **retained as an optional fallback**; (2) the runner
  still **reads governance from the store's own scripts/refs** (no logic duplicated — it just reads a
  local copy instead of a submodule); (3) versions are stored **verbatim, extracted offline** from the
  ai-skills git tags (`git archive`, no network), so the copy is a faithful snapshot, not a fork;
  (4) `runner check` still flags when a newer version exists.

## Motivation
Run fully offline with both contract versions available locally, and let the runner auto-select the
right version per project lock.

## Decisions & trade-offs
| Decision | Options | Why this choice |
|----------|---------|-----------------|
| Store location | ai-skills side vs **inside runner** | User chose inside the runner (`skills/`) for a self-contained offline runner (overrides §7; recorded above) |
| Submodule | remove vs **keep as fallback** | User: keep `.gitmodules` as an optional, default-not-pulled fallback |
| Version selection | config-fixed vs **auto by project lock** | User: resolve the store version from the project's `.sdlc-lock.json` major.minor |
| Extraction | online clone vs **offline `git archive`** | User: never fetch online; extract verbatim from local tags |
| Layout | flat vs per-version dirs | `skills/v1.0.0/`, `skills/v1.1.0/` (each a full skill root with SKILL.md/references/scripts/assets) |

## Modification Guide
### Goal
Vendor v1.0.0 + v1.1.0 into `skills/`, add a `skillstore` module to list/resolve versions, make skill-
path resolution store-aware (auto by lock; `--skill-path` override; submodule fallback), and keep
`check` working against the store. Offline only.

### Impact scope
| Affected item | Type | Change | Knock-on effect |
|---------------|------|--------|-----------------|
| `skills/v1.0.0/`, `skills/v1.1.0/` | NEW directory/data | Vendored skill roots (verbatim from tags) | Primary offline source; overrides §7 (recorded) |
| `src/ai_sdlc_runner/skillstore.py` | NEW logical/design | `store_versions`/`resolve_path`/`latest_version`/`detect` | Version resolution + store update detection |
| `src/ai_sdlc_runner/cli.py` | logical | `_resolve_skill_path` becomes store-aware (by project lock); `check` lists store versions | Run/check/status/dashboard use the store automatically |
| `config/runner.yaml` | data | Add `skill_store: "./skills"`; note skill_path/submodule are fallback | Resolution order: override → store(by lock) → skill_path/submodule |
| `.gitmodules` | directory | Note: optional fallback, default not pulled | — |
| `docs/ai-guideline.md` | docs | Record the override under conventions | Governance trace |
| `docs/structure/*` , `README.md` | docs | Document the store | — |
| `tests/test_skillstore.py` | logical | NEW: version list/resolve/detect | Acceptance evidence |

### Modification steps
1. Extract `skills/ai-sdlc` from tags `ai-sdlc-v1.0.0` / `ai-sdlc-v1.1.0` into `skills/v1.0.0` / `skills/v1.1.0` (offline `git archive`).
2. Add `skillstore.py` (list/resolve/latest/detect-update against the store).
3. Make `cli._resolve_skill_path(config, override, project_dir)` store-aware (lock → version dir; else config-expected; else latest; else submodule path).
4. `config/runner.yaml`: add `skill_store`. Update `.gitmodules` note. `check` lists store versions.
5. Record the override in the Guideline; sync structure docs + README; add tests.

### Risk & rollback
- Risk: medium — skill sourcing changed + guardrail relaxed; mitigated by verbatim offline extraction,
  retained submodule fallback, and no duplicated governance logic (the runner reads the store's own
  scripts/refs). Gates and V1 lockdown unchanged.
- Rollback: delete `skills/v*`, remove `skillstore.py` + `skill_store` config, revert `_resolve_skill_path`; submodule path still works.

## Structure change summary
Adds a vendored, offline, multi-version skill store inside the runner and a `skillstore` resolver;
skill-path resolution is now store-first (auto by lock) with submodule fallback. The override of the
reference-not-copy guardrail is explicitly recorded here and in the Guideline.
